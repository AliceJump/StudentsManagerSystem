# 07-开发规范

## 命名规范实际情况

- 类名使用 PascalCase，例如 `StudentRepository`、`ScoreService`。
- 方法名使用 PascalCase，例如 `GetAll`、`Search`、`ValidateCredentials`。
- 私有字段多使用 camelCase，例如 `studentService`、`currentPage`。
- XAML 控件命名使用前缀，例如 `txtStudentNo`、`cmbSemester`、`dataGrid`。

## 文件组织规范

- 模型放在 `Models`。
- 页面和窗口放在 `Views/<模块>`。
- Repository 放在 `Data/Repositories`，实际使用 EF Core SQLite。
- Service 放在 `Services`。
- 通用工具放在 `Common`。
- 文档放在 `docs`。

## ViewModel 规范实际情况

- `MainViewModel` 实际用于主窗口绑定标题和时间。
- `StudentArchiveViewModel` 是骨架，尚未接入页面。
- 多数页面仍通过 code-behind 事件处理，不是完整 MVVM。

## Service 规范实际情况

- 已有 `StudentService`、`ScoreService`、`BasicDataService`。
- Service 负责输入校验、重复检查、调用 Repository、记录日志。
- Service 返回 `ServiceResult` 表达成功或失败。
- 学籍模块尚未统一使用 Service。

## Repository 规范实际情况

- Repository 每个方法内部创建 `StudentsManagerDbContext`。
- 查询使用 `AsNoTracking`。
- 搜索使用 `EF.Functions.Like`。
- 增删改直接调用 EF Core `Add`、`Update`、`Remove`、`SaveChanges`。

## 数据来源规范

- 默认数据只能在 `DatabaseInitializer` 中维护。
- 可变业务选项必须从数据库读取。
- XAML 不得直接硬编码院系、专业、班级、课程、学期、状态等业务选项。
- 允许硬编码的占位选项限于“全部”“请选择”“未选择”“无数据”。

## 异常处理规范实际情况

- 启动阶段由 `App.OnStartup` 捕获数据库初始化异常。
- UI 未处理异常通过 `DispatcherUnhandledException` 处理。
- 导入导出操作局部 try/catch 并弹窗。
- Repository 层多数方法未统一包装异常。

## 日志规范实际情况

- 日志工具为 `Common/AppLogger.cs`。
- 通过 `AppLogger.Info/Warn/Error` 写入文本日志。
- 日志位置：运行目录 `Logs/StudentsManagerSystem.log`。
- 日志超过 1MB 自动轮转，最多保留 10 个归档文件；系统提供日志查看页面。

## 数据库规范

- 统一初始化入口：`DatabaseInitializer.Initialize()`。
- 学生业务身份使用 `StudentNo`。
- 学生档案子模块不得展示 `StudentId`。
- 当前没有 EF Migration，结构补齐由初始化模块处理。
