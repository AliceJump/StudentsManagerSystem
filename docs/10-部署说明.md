# 10-部署说明

## 环境要求

- Windows 10/11
- .NET 8.0 SDK 或 Runtime
- Visual Studio 2022 或 Visual Studio Code 可用于开发

## 本地构建

在仓库根目录执行：

```bash
dotnet build "StudentsManagerSystem/StudentsManagerSystem.csproj"
```

如果运行中的 exe 占用导致 apphost 复制警告，可关闭程序，或临时使用：

```bash
dotnet build "StudentsManagerSystem/StudentsManagerSystem.csproj" /p:UseAppHost=false
```

## 本地运行

```bash
dotnet run --project "StudentsManagerSystem/StudentsManagerSystem.csproj"
```

启动后流程：

1. 初始化 SQLite 数据库。
2. 显示登录窗口。
3. 登录成功后进入主窗口。

## 默认配置

配置文件：[`StudentsManagerSystem/App.config`](../StudentsManagerSystem/App.config)

- 数据库文件：`StudentsManagerSystem.db`
- 管理员用户名：`admin`
- 管理员密码：`Admin@123`

## 数据库文件位置

[`DatabaseConfiguration`](../StudentsManagerSystem/Data/DatabaseConfiguration.cs#L11) 会将相对路径解析到程序运行目录。开发运行时通常位于输出目录，例如：

```text
StudentsManagerSystem/bin/Debug/net8.0-windows/StudentsManagerSystem.db
```

## 日志位置

日志写入运行目录下：

```text
Logs/StudentsManagerSystem.log
```

## 发布建议

可使用 `dotnet publish` 生成发布目录：

```bash
dotnet publish "StudentsManagerSystem/StudentsManagerSystem.csproj" -c Release
```

## 注意事项

- 当前没有安装包脚本。
- 当前没有自动数据库迁移流程，仅在启动时 `EnsureCreated` 并执行轻量补列。
- 发布前应确认是否要随包携带已有 SQLite 数据库文件。
- 首次启动会初始化种子数据。
