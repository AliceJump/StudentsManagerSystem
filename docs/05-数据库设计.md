# 05-数据库设计

## 数据库与 ORM

- 数据库：SQLite
- ORM：Entity Framework Core
- DbContext：`StudentsManagerSystem/Data/StudentsManagerDbContext.cs`
- 初始化入口：`StudentsManagerSystem/Data/DatabaseInitializer.cs`
- 连接配置：`StudentsManagerSystem/App.config`

## DbSet 清单

- `Students`
- `FamilyInfos`
- `RewardRecords`
- `PunishmentRecords`
- `HealthRecords`
- `StudentRegistrations`
- `StatusChangeRecords`
- `ScholarshipInfos`
- `GraduationInfos`
- `Scores`
- `Courses`
- `Departments`
- `Majors`
- `Classes`
- `Users`
- `LookupOptions`

## 关键唯一键

- `Students.StudentNo`：学号，学生业务唯一键。
- `Students.IdCard`：身份证号唯一。
- `Courses.CourseNo`：课程编号唯一。
- `Departments.DepartmentNo`：院系编号唯一。
- `Majors.MajorNo`：专业编号唯一。
- `Classes.ClassNo`：班级编号唯一。
- `Users.Username`：用户名唯一。
- `LookupOptions.Category + LookupOptions.Value`：字典项唯一。

## ER 说明

```text
Students(StudentNo unique)
├── FamilyInfos(StudentNo)
├── RewardRecords(StudentNo)
├── PunishmentRecords(StudentNo)
├── HealthRecords(StudentNo)
├── StudentRegistrations(StudentNo)
├── StatusChangeRecords(StudentNo)
├── ScholarshipInfos(StudentNo)
├── GraduationInfos(StudentNo)
└── Scores(StudentNo, CourseNo)

Courses(CourseNo) ── Scores(CourseNo)
Departments(DepartmentName) ── Majors(DepartmentName) ── Classes(MajorName)
LookupOptions(Category, Value) ── UI business option lists
Users ── LoginWindow / role check
```

## 外键关系实际情况

当前代码没有配置 EF Core 外键约束。多表关联主要依赖业务字段：

- 学生相关业务优先使用 `StudentNo`。
- 部分旧实体仍保留 `StudentId` 字段用于兼容既有数据。
- 学生档案家庭、奖励、处分、体检已经补充 `StudentNo` 并按 `StudentNo` 展示和清理。

## 主要表结构摘要

### Students

字段：`Id`、`StudentNo`、`Name`、`Gender`、`BirthDate`、`IdCard`、`Nation`、`PoliticalStatus`、`PhoneNumber`、`Email`、`Address`、`Department`、`Major`、`Class`、`EnrollmentDate`、`Photo`。

### 学生档案子表

- `FamilyInfos`：`StudentNo`、`RelationName`、`Relationship`、`PhoneNumber`、`WorkUnit`、`Address`。
- `RewardRecords`：`StudentNo`、`RewardDate`、`RewardType`、`RewardLevel`、`RewardReason`、`RewardUnit`。
- `PunishmentRecords`：`StudentNo`、`PunishmentDate`、`PunishmentType`、`PunishmentLevel`、`PunishmentReason`、`CancelDate`、`Status`。
- `HealthRecords`：`StudentNo`、`CheckDate`、`Height`、`Weight`、`BloodType`、`Vision`、`HealthStatus`、`Remarks`。

### 学籍表

- `StudentRegistrations`：注册日期、学年、学期、状态。
- `StatusChangeRecords`：变动日期、变动类型、原信息、新信息、审批状态。
- `ScholarshipInfos`：奖助类型、等级、金额、发放日期、状态。
- `GraduationInfos`：毕业日期、毕业类型、学位类型、证书号。

### 成绩表

- `Courses`：课程基础数据。
- `Scores`：学生课程成绩，导入合并依据为 `StudentNo + AcademicYear + Semester + CourseNo`。

### 系统表

- `Users`：登录账号和角色。
- `LookupOptions`：性别、政治面貌、学期、审批状态等可变字典。

## 初始化规则

- 默认数据只能放在 `DatabaseInitializer`。
- XAML 和业务代码不得硬编码院系、专业、班级、课程、学期、状态等业务选项。
- 可变业务选项从业务表或 `LookupOptions` 读取。
