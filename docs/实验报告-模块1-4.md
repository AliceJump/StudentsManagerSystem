```csharp
public bool ValidateCredentials(string username, string password, out string? displayName, out string? role)
{
    displayName = null;
    role = null;
    username = username.Trim();
    if (string.IsNullOrWhiteSpace(username) || string.IsNullOrWhiteSpace(password))
        return false;

    using var context = StudentsManagerDbContextFactory.CreateDbContext();
    var user = context.Users.FirstOrDefault(item => item.Username == username && item.IsActive);
    if (user == null)
        return false;

    var valid = string.Equals(user.PasswordHash, ComputeHash(password), StringComparison.OrdinalIgnoreCase);
    if (!valid)
        return false;

    displayName = user.DisplayName;
    role = user.Role;
    user.LastLoginAt = DateTime.Now;
    context.Users.Update(user);
    context.SaveChanges();
    return true;
}

private static string ComputeHash(string input)
{
    using var sha = SHA256.Create();
    var bytes = Encoding.UTF8.GetBytes(input);
    var hashed = sha.ComputeHash(bytes);
    return Convert.ToHexString(hashed).ToLowerInvariant();
}
```

#### 问题1（文字描述）

**登录失败 — 用户名或密码错误**

在登录窗口输入错误的用户名或密码，系统提示"用户名或密码错误"，登录被拒绝。

#### 问题1（错误注入方式）

**数据错误注入**

通过 SQLite 数据库直接修改用户密码哈希值，使正确密码无法通过校验：

```sql
UPDATE Users SET PasswordHash = 'faultyhashvalue1234567890' WHERE Username = 'admin';
```

或者更简单的模拟方式：在登录时直接输入错误密码。

#### 问题1（复现步骤）

1. 启动程序，显示登录窗口
2. 在"用户名"输入框输入 `admin`
3. 在"密码"输入框输入 `WrongPassword123`（正确的密码应为 `Admin@123`）
4. 点击"登录"按钮

#### 问题1（截图）

**说明**：截图登录窗口，聚焦在红色错误提示文字"用户名或密码错误"上。不需要输入框中的具体内容，但需要看到错误信息显示在输入框下方。

#### 问题1（原因分析）

**分析**：`UsersRepository.ValidateCredentials()` 方法中，使用 SHA256 计算输入密码的哈希值，与数据库中存储的 `PasswordHash` 字段进行比对：

```csharp
var valid = string.Equals(user.PasswordHash, ComputeHash(password), StringComparison.OrdinalIgnoreCase);
if (!valid)
    return false;
```

由于输入密码 `WrongPassword123` 的 SHA256 哈希值与数据库中的 `Admin@123` 的哈希值不匹配，`ComputeHash(password)` 返回的字符串与 `user.PasswordHash` 不同，导致 `valid = false`，方法返回 `false`，`LoginWindow` 收到失败结果后显示"用户名或密码错误"。

**涉及模块**：模块一（登录相关功能）

**涉及代码**：
- [`Views/Login/LoginWindow.xaml.cs`](../StudentsManagerSystem/Views/Login/LoginWindow.xaml.cs#L22) 第 22-33 行（登录按钮事件）
- [`Data/Repositories/UsersRepository.cs`](../StudentsManagerSystem/Data/Repositories/UsersRepository.cs#L12) 第 12-39 行（ValidateCredentials 方法）
- [`Data/Repositories/UsersRepository.cs`](../StudentsManagerSystem/Data/Repositories/UsersRepository.cs#L49) 第 49-55 行（ComputeHash 方法）

#### 问题1（解决方案）

**方案**：输入正确的密码即可。管理员账号 `admin` 的默认密码在 `App.config` 中配置：

```xml
<add key="AdminPassword" value="Admin@123" />
```

正确操作：用户名输入 `admin`，密码输入 `Admin@123`，点击登录。

如果需要通过数据库恢复，可以重新运行程序让 `DatabaseInitializer.SeedUsers()` 重新创建用户（删除数据库文件后重启即可），或者在 SQLite 中执行：

```sql
-- 计算 Admin@123 的 SHA256 哈希值
-- 哈希值: 0b1c7bf7a9a6c2d4e8f3a5b7c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d
UPDATE Users SET PasswordHash = '0b1c7bf7a9a6c2d4e8f3a5b7c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d' WHERE Username = 'admin';
```

#### 问题1（解决方案-正确截图）

**说明**：使用正确的用户名 `admin` 和密码 `Admin@123` 登录，弹出提示框显示"登录成功：系统管理员"，点击确定后进入主窗口，标题栏显示"学生管理系统 - 系统管理员"。

#### 问题2（文字描述）

**登录失败 — 用户被禁用**

管理员将用户账号设置为停用状态后，该用户无法登录系统。

#### 问题2（错误注入方式）

**数据错误注入**

在 SQLite 数据库中直接修改 `IsActive` 字段：

```sql
UPDATE Users SET IsActive = 0 WHERE Username = 'teacher';
```

#### 问题2（复现步骤）

1. 使用 SQLite 数据库工具（如 DB Browser for SQLite）打开 `StudentsManagerSystem.db`
2. 执行 SQL：`UPDATE Users SET IsActive = 0 WHERE Username = 'teacher';`
3. 关闭数据库工具
4. 启动程序，显示登录窗口
5. 在"用户名"输入框输入 `teacher`
6. 在"密码"输入框输入 `Teacher@123`
7. 点击"登录"按钮

#### 问题2（截图）

**说明**：截图登录窗口，显示红色错误提示"用户名或密码错误"。注意尽管密码正确，但由于 IsActive=0，系统表现为"用户名或密码错误"（不提示具体是账户禁用还是密码错误，这是安全设计）。

#### 问题2（原因分析）

**分析**：在 `UsersRepository.ValidateCredentials()` 中，查询用户时使用了 `IsActive` 过滤条件：

```csharp
var user = context.Users.FirstOrDefault(item => item.Username == username && item.IsActive);
```

当 `IsActive = 0`（即 `false`）时，`FirstOrDefault` 不会返回该用户，`user == null`，方法直接返回 `false`，登录失败。

这是安全设计——不区分"用户不存在"和"用户被禁用"，防止攻击者通过错误信息判断用户是否存在。

**涉及模块**：模块一（登录相关功能）+ 模块三（权限相关功能）

**涉及代码**：[`Data/Repositories/UsersRepository.cs`](../StudentsManagerSystem/Data/Repositories/UsersRepository.cs#L19) 第 19 行

**涉及数据库表**：`Users` 表的 `IsActive` 字段

#### 问题2（解决方案）

**方案**：恢复用户的启用状态。在 SQLite 中执行：

```sql
UPDATE Users SET IsActive = 1 WHERE Username = 'teacher';
```

之后重启程序，使用 `teacher` / `Teacher@123` 即可正常登录。

#### 问题2（解决方案-正确截图）

**说明**：修复后使用 `teacher` / `Teacher@123` 登录成功，弹出提示框"登录成功：普通教师"，进入主窗口后标题栏显示"学生管理系统 - 普通教师"，且"基础数据管理"按钮为禁用状态（灰色）。

public static string ConnectionString
{
    get
    {
        var configured = ConfigurationManager.ConnectionStrings[ConnectionStringName]?.ConnectionString;
        if (!string.IsNullOrWhiteSpace(configured))
        {
            var builder = new SqliteConnectionStringBuilder(configured);
            if (string.IsNullOrWhiteSpace(builder.DataSource))
                builder.DataSource = ResolveDatabaseFileName();
            else if (!Path.IsPathRooted(builder.DataSource))
                builder.DataSource = Path.GetFullPath(Path.Combine(AppContext.BaseDirectory, builder.DataSource));
            builder.Mode = SqliteOpenMode.ReadWriteCreate;
            return builder.ToString();
        }
        return new SqliteConnectionStringBuilder
        {
            DataSource = ResolveDatabaseFileName(),
            Mode = SqliteOpenMode.ReadWriteCreate
        }.ToString();
    }
}
