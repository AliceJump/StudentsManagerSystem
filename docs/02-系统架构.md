# 02-系统架构

## 实际架构模式

### 已实际采用

- 分层架构：UI、Service、Repository、Data、Models 分目录组织。
- Repository Pattern：[`Data/Repositories/*Repository.cs`](../StudentsManagerSystem/Data/Repositories/) 封装数据访问。
- Service 层：[`Services/StudentService.cs`](../StudentsManagerSystem/Services/StudentService.cs#L8)、[`StudentStatusService.cs`](../StudentsManagerSystem/Services/StudentStatusService.cs#L10)、[`ScoreService.cs`](../StudentsManagerSystem/Services/ScoreService.cs#L10)、[`BasicDataService.cs`](../StudentsManagerSystem/Services/BasicDataService.cs#L8) 承担业务校验、日志和保存逻辑。
- 部分 MVVM：[`ViewModels/MainViewModel.cs`](../StudentsManagerSystem/ViewModels/MainViewModel.cs#L11) 绑定主窗口标题和时间；但多数业务页面仍使用 code-behind 事件处理。

### 未实际采用或仅部分体现

- MVC：未采用。
- Clean Architecture：未严格采用，项目没有独立 Domain/Application/Infrastructure 项目边界。
- DDD：未采用聚合、领域服务、值对象等结构。
- CQRS：未区分 Command/Query 模型。
- Event Driven：未使用领域事件或消息总线。

## 分层说明

### UI 层

- [`MainWindow.xaml`](../StudentsManagerSystem/MainWindow.xaml) 和 [`Views/**/*.xaml`](../StudentsManagerSystem/Views/)
- 处理控件布局、事件、弹窗、DataGrid 展示。
- 部分页面直接调用 Service 或 Repository。

### Application 层

- [`Services/*.cs`](../StudentsManagerSystem/Services/)
- 当前覆盖学生、学籍、成绩、基础数据。
- 职责包括业务校验、调用 Repository、日志记录、导入合并规则。

### Domain 层

- [`Models/*.cs`](../StudentsManagerSystem/Models/)
- 当前是贫血模型，主要承载属性，没有领域行为。

### Data / Infrastructure 层

- [`Data/StudentsManagerDbContext.cs`](../StudentsManagerSystem/Data/StudentsManagerDbContext.cs#L9)
- [`Data/DatabaseInitializer.cs`](../StudentsManagerSystem/Data/DatabaseInitializer.cs#L14)
- [`Data/Repositories/*Repository.cs`](../StudentsManagerSystem/Data/Repositories/)
- [`Common/AppLogger.cs`](../StudentsManagerSystem/Common/AppLogger.cs#L9)
- [`Common/CsvExportHelper.cs`](../StudentsManagerSystem/Common/CsvExportHelper.cs#L11)

## 核心调用方向

典型方向：

```text
XAML / Code-behind
-> Service
-> Repository
-> StudentsManagerDbContext
-> SQLite
```

学籍窗口也已进入 Service 调用链路。

## 启动流程

1. [`App.OnStartup`](../StudentsManagerSystem/App.xaml.cs#L18) 写入启动日志。
2. [`DatabaseInitializer.Initialize()`](../StudentsManagerSystem/Data/DatabaseInitializer.cs#L16) 创建数据库、表和初始化数据。
3. 显示 [`LoginWindow`](../StudentsManagerSystem/Views/Login/LoginWindow.xaml.cs#L9)。
4. 登录成功后保存 [`App.CurrentUserDisplayName`](../StudentsManagerSystem/App.xaml.cs#L16) 和 [`App.CurrentUserRole`](../StudentsManagerSystem/App.xaml.cs#L14)。
5. 创建并显示 [`MainWindow`](../StudentsManagerSystem/MainWindow.xaml.cs#L19)。
6. [`MainWindow`](../StudentsManagerSystem/MainWindow.xaml.cs#L19) 根据角色控制基础数据入口。

## 架构风险

- `Views` 中仍有较多业务事件逻辑，MVVM 不完整。
- 部分业务页面仍以事件驱动为主，ViewModel 可继续深化。
