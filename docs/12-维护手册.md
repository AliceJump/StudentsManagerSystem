# 12-维护手册

## 日常检查

- 构建：`dotnet build "StudentsManagerSystem/StudentsManagerSystem.csproj" /p:UseAppHost=false`
- 文档：确认 README、功能文档、数据库文档与代码一致。
- 数据库：确认初始化数据只在 [`DatabaseInitializer`](../StudentsManagerSystem/Data/DatabaseInitializer.cs#L14)。
- XAML：确认没有硬编码业务下拉项。

## 常见问题

### 构建时 exe 被占用

现象：`StudentsManagerSystem.exe` 无法复制。

处理：关闭正在运行的程序，或使用 `/p:UseAppHost=false` 构建。

### 登录失败

检查：
- `App.config` 中 `AdminUsername`、`AdminPassword`。
- SQLite 数据库是否已初始化。
- `Users` 表中用户是否 `IsActive = true`。

### 数据库结构未更新

当前没有 Migration。轻量补列逻辑在 `DatabaseInitializer` 中执行。若新增表或字段，应同步更新 DbContext 和初始化模块。

### 下拉框无数据

检查：
- 对应业务表是否有数据。
- `LookupOptions` 是否初始化。
- 页面是否调用 Service 加载 ItemsSource。

### 日志不存在

- 日志在运行目录 `Logs/StudentsManagerSystem.log`。如果目录不存在，[`AppLogger`](../StudentsManagerSystem/Common/AppLogger.cs#L9) 会自动创建。

## 维护约束

- 不要在 View、ViewModel、Service、Repository 中写默认数据。
- 不要在 XAML 写死业务选项。
- 不要让学生档案子模块展示 `StudentId`。
- 不要绕过 Repository 直接在 UI 操作数据库。

## 数据备份

SQLite 数据库是单文件。备份时复制运行目录下的 `StudentsManagerSystem.db`。

## 日志清理

当前没有自动清理策略。需要人工删除或后续实现滚动日志。

## 发布前检查清单

- 构建通过。
- README 与 docs 更新。
- 默认管理员密码是否已按部署环境修改。
- 数据库文件是否需要随包发布。
- `bin`、`obj` 等构建产物不应手工编辑。

## 技术债清单

- ViewModel 使用仍可继续深化，当前主窗口已使用 [`MainViewModel`](../StudentsManagerSystem/ViewModels/MainViewModel.cs#L11)。
- 当前已有测试项目，需继续补业务 Service 和 Repository 测试。
- 当前已有 EF Core Migration 基线，后续结构变更应优先使用 Migration。
- 当前已有日志轮转和日志查看页面，后续可补日志级别筛选。
